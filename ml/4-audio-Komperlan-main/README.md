# Лабораторная 4. Классификация аудио

Проект классифицирует персонажей Dota 2 по голосовым репликам. Для каждого персонажа собираются MP3-файлы, затем аудио переводится в mel-спектрограммы и подается в LSTM-классификатор.

## Файлы

- `AudioLab.ipynb` - основной ноутбук с подготовкой данных, моделью, обучением, тестированием и визуализацией эмбеддингов.
- `audioParser.py` - скрипт для скачивания голосовых реплик с Dota 2 Fandom.
- `LICENSE` - лицензия проекта.

## Датасет

В работе используются реплики пяти персонажей:

- `Crystal_Maiden`;
- `Invoker`;
- `Silencer`;
- `Templar_Assassin`;
- `Winter_Wyvern`.

В сохраненном запуске ноутбука было найдено `1668` аудиофайлов:

- train: `1167`;
- validation: `167`;
- test: `334`.

В ноутбуке путь к данным задан как:

```python
dataset_path = "/content/drive/MyDrive/AudioDataSet"
```

При локальном запуске его нужно заменить на путь к папке с подпапками персонажей.

## Парсер аудио

`audioParser.py` открывает страницы вида:

```text
https://dota2.fandom.com/wiki/<Hero_Name>/Responses
```

Скрипт использует Selenium в headless-режиме, ищет теги `audio`, достает ссылки на MP3 и сохраняет файлы в папки с именами героев.

## Подготовка признаков

Для каждого MP3:

- аудио загружается через `librosa`;
- длительность приводится к `3` секундам;
- строится mel-спектрограмма;
- спектрограмма переводится в dB и нормализуется.

Ключевые параметры:

- `sample_rate`: `22050`;
- `duration`: `3.0`;
- `n_mels`: `128`;
- `n_fft`: `2048`;
- `hop_length`: `512`.

## Модель

В ноутбуке реализованы:

- собственная `LSTMCell`;
- многослойный `LSTMLayer`;
- `DotaVoiceClassifier` с LSTM и полносвязной классификационной головой.

Параметры обучения:

- `batch_size`: `40`;
- `epochs`: `30`;
- `learning_rate`: `0.005`;
- `hidden_size`: `60`;
- `num_layers`: `4`;
- `dropout`: `0.3`.

## Результаты

В сохраненном запуске:

- число параметров модели: `140 933`;
- test loss: `0.2285`;
- test accuracy: `0.9281`.

После обучения из последнего слоя LSTM извлекаются эмбеддинги размера `60`, затем они визуализируются в 2D с помощью t-SNE. Визуализация показывает, что голоса группируются по персонажам.
